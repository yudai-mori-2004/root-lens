# Task 13: UX向上

## 目的

日常利用に耐えるUXを整備する。機能追加ではなく、既存フローの磨き込み。

## 項目

### 13.1 プレビューページ: 3点メニュー + コンテンツ削除

共有ボタンの右に `...` メニューを追加し、以下のアクションを提供:

- **コンテンツ削除**: サーバー上のページを非公開化（Supabase status → 'deleted'）
- 将来: リンクコピー、問題を報告、等

### 13.2 ギャラリー: 公開済みコンテンツの表示

ギャラリー内のサムネイルに、既にTitle Protocolに登録済み（公開済み）のコンテンツであることを示すマーカーを表示する。

- オレンジのチェックマークをサムネイル角に表示
- 判定方法: C2PA署名済みファイルの content_hash をサーバーの pages テーブルと照合

### 13.3 設定画面: 言語設定

- 日本語 / English の切り替えUI
- `expo-localization` の自動検出 + 手動オーバーライド
- 設定値は AsyncStorage に永続化

### 13.4 ホーム: プロフィール表示の改善

- Instagram風のグリッドレイアウトに寄せる
- アバター + ユーザー名 + 投稿数の上部表示
- 公開済みコンテンツのグリッド一覧

### 13.5 設定画面: 端末情報 — TEE署名ステータス

設定 > 端末情報に、現在の署名能力を表示:

- TEE鍵ペアの存在確認（Secure Enclave / StrongBox）
- デバイス証明書の有効期限
- 署名テスト（ダミーデータに署名してみて成功するか）
- Hardware Attestation のサポート状況

ユーザーが「自分の端末は本当にTEEで署名できる状態にあるのか」を確認できるようにする。

### 13.6 アプリアイコン

- RootLensのブランドアイコンを作成・設定
- Android adaptive icon (foreground + background)
- iOS App Icon (1024x1024 + 各サイズ)
- Expo の `icon` / `adaptiveIcon` 設定

### 13.7 複数コンテンツ対応

現状、編集画面で複数枚選択→登録しても、公開ページには1枚目しか表示されない。
ホーム（PublishedGalleryScreen）でも複数枚ある投稿の区別がつかない。

- **公開ページ**: 複数コンテンツをスワイプ/カルーセルで表示
- **サーバー**: 1ページに複数 content を紐づける構造は既にある（pages → contents 1:N）が、
  公開パイプライン（PublishingScreen）が1枚目のみ処理している
- **ホーム**: 複数枚の投稿にはスタックアイコン等のインジケーターを表示
- **プレビュー**: 複数コンテンツのスワイプ対応

### 13.8 動画対応

現状、動画のC2PA署名・Title Protocol登録・公開ページ表示が未対応または不完全。

- **署名**: 動画（MP4/MOV）のC2PA署名は c2pa-rs が対応しているが、
  実際のフローで正しく動作するか要検証
- **登録**: PublishingScreen のパイプラインが画像前提（ImageManipulator でリサイズ等）。
  動画の場合はサムネイル抽出 + 動画本体のアップロードが必要
- **公開ページ**: 動画プレイヤーの表示（`<video>` タグ or ストリーミング）
- **ホーム**: 動画コンテンツには再生アイコンをオーバーレイ表示
