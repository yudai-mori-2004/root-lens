# Task 02: カメラ撮影 → 編集 → 証明フローの一気通貫

## 目的

撮影から証明（公開）までのメインフローをUI上で一通り動かせる状態にする。
Instagram的なUXをベースとし、モダンなUIで実装する。
C2PA署名・Title Protocol連携・サーバー通信はスコープ外。

## UX設計

### カメラ（Instagram参照）
- フッター中央ボタン → フルスクリーンモーダルでカメラ起動（タブバー非表示）
- 写真/動画モード切替（スワイプ or タブ。動画は後回しでOKだがUI枠は用意）
- 撮影ボタン、フロント/リアカメラ切替、フラッシュ
- 何枚でも連続撮影可能（撮影後すぐ編集には入らない）
- 撮影した写真は端末フォトライブラリに保存
- 閉じるボタンでタブ画面に戻る

### 端末ギャラリー（右タブ）
- 端末内の全画像を3列グリッド表示
- タップで複数選択（チェックマーク + 選択番号）
- 将来: C2PA署名付きのみ選択可能 + バッジ表示。現時点では全画像選択可能
- 選択中はヘッダーに「次へ」ボタン表示

### 編集画面
- 選択した画像を1枚ずつ編集（ページネーション形式）
- **ヘッダー**: ← (undo) / → (redo) / 「次へ」or「証明」ボタン
  - 複数枚: 最後の1枚以外は「次へ」、最後の1枚は「証明」
  - 1枚のみ: 「証明」
- **メイン**: 画像プレビュー
- **フッター**: 編集ツール（クロップ等）のみ。公開ボタンや下書きボタンは置かない
- **自動保存**: 操作するたびに自動保存。undo/redo で操作履歴を行き来可能

### 証明後
- ローディング画面（登録処理中のモック）
- 完了後、WebViewで仮のページを表示（将来はrootlens.ioのiframe）

## UIルール
- 絵文字アイコン禁止。@expo/vector-icons（Ionicons / MaterialIcons）を使用
- モダンでクリーンなデザイン
